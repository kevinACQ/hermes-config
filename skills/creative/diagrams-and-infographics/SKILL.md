# Diagrams & Infographics

## Overview

Umbrella for explanatory visuals: dark SVG architecture/cloud diagrams, hand-drawn Excalidraw diagrams, and structured infographic layouts/styles.

## When to Use

- User asks for architecture, flow, sequence, cloud, or system diagrams.
- User asks for a hand-drawn sketch/whiteboard aesthetic.
- User asks to turn dense information into an infographic, visual map, comparison, or layout.
- User wants reusable templates/styles for diagrams.

## Modes

### Architecture Diagrams

Use precise nodes, edges, labels, trust boundaries, data flows, and deployment zones. Prefer SVG/HTML for sharable artifacts.

### Excalidraw

Use when the desired feel is hand-drawn, collaborative, or whiteboard-like. Export valid Excalidraw JSON and verify upload/render when possible.

### Infographics

Use labeled content hierarchy, layout pattern, and visual style. Convert source material into structured claims before designing.

## Workflow

1. Identify the visual question: architecture, sequence, comparison, journey, hierarchy, or explainer.
2. Extract entities, relationships, and hierarchy.
3. Pick format: SVG/HTML, Excalidraw JSON, or infographic prompt/template.
4. Render/validate the artifact when possible.
5. Report the artifact path/URL.

## Verification Checklist

- [ ] Artifact file exists and is valid for its format.
- [ ] Labels are readable and relationships are unambiguous.
- [ ] Diagram answers the user's question rather than decorating it.
- [ ] Any generated visual has a deliverable path/URL.
